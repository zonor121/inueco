## Практическое задание

**Тема:** Работа с табличными данными в pandas. Первичный анализ и индексация
**Датасет:** `insurance.csv` (Kaggle, медицинские страховые расходы) https://www.kaggle.com/datasets/mirichoi0218/insurance 
https://www.kaggle.com/datasets/mosapabdelghany/medical-insurance-cost-dataset
### Цель практической работы

Ваша задача —  поработать с `DataFrame`: загружать данные, понимать их структуру, обращаться к строкам и столбцам, работать с индексами и выполнять первичный осмотр датасета.


### Исходные данные

Вы работаете с датасетом **insurance.csv**, который содержит 1338 строк и 7 столбцов:

* `age` — возраст застрахованного
* `sex` — пол
* `bmi` — индекс массы тела
* `children` — количество детей
* `smoker` — курит или нет
* `region` — регион проживания
* `charges` — стоимость медицинских услуг

Файл необходимо скачать с Kaggle и положить рядом с ноутбуком (лучше .ipynb) или Python-скриптом. (уже в директории)

## Задание

### Шаг 1. Подготовка окружения

1. Создайте новый Jupyter Notebook:
   * либо в Google Colab,
   * либо локально в VS Code.

2. Подключите библиотеку `pandas`.
3. Убедитесь, что версия Python не ниже 3.9.

Результат шага — ноутбук открывается, код выполняется без ошибок.

### Шаг 2. Загрузка данных из CSV-файла

1. Считайте файл `insurance.csv` в `DataFrame`.
2. Сохраните результат в переменную с осмысленным именем.
3. Выведите первые 10 строк таблицы.
4. Выведите последние 5 строк таблицы.

### Шаг 3. Первичный осмотр данных

Для загруженного `DataFrame` выполните:

1. Просмотр общей информации о таблице:
   * количество строк,
   * количество столбцов,
   * типы данных.

2. Проверку наличия пропущенных значений.
3. Просмотр названий всех столбцов.
4. Определение, какие столбцы являются числовыми, а какие категориальными.

После выполнения этого шага вы должны **словами** (в markdown-ячейке) ответить:

* есть ли пропуски в данных,
* какие типы данных использует pandas,
* корректны ли эти типы с точки зрения смысла данных.

### Шаг 4. Работа с индексом DataFrame

1. Выведите текущий индекс таблицы.
2. Объясните, что именно сейчас используется в качестве индекса.
3. Установите в качестве индекса столбец `age`.
4. Снова выведите первые 5 строк таблицы и обратите внимание на изменения.
5. Верните индекс обратно к стандартному целочисленному.

### Шаг 5. Доступ к данным

Выполните следующие операции:

1. Получите столбец `charges` тремя разными способами доступа.
2. Получите одну строку по индексу.
3. Получите диапазон строк (например, с 10 по 20).
4. Получите подтаблицу, содержащую только столбцы `age`, `bmi`, `charges`.
5. Получите данные только для курящих людей (`smoker == "yes"`).

Каждую операцию сопровождайте кратким комментарием, **что именно вы получили и почему**.

### Шаг 6. Простые изменения данных

1. Переименуйте столбец `charges` в `insurance_cost`.
2. Создайте новый столбец `bmi_category`, в котором:
   * `bmi < 18.5` → `underweight`
   * `18.5–24.9` → `normal`
   * `25–29.9` → `overweight`
   * `>= 30` → `obese`

3. Проверьте, что новый столбец добавился корректно.
4. Верните исходное название столбца `charges`.
